# 設計: らくしふ客数ベースのLE/REQ計算（配布版 v2.2）

2026-07-18 設計。目的: 配布版をスプレッドシート非依存にする。
LE＝らくしふの「修正客数」そのもの、REQ＝負担率モデルによる計算値とし、
係数はすべて設定画面で調整可能にする。

## 1. データソース（調査済み・実機確認済み）

「修正客数」「前年客数」は会社カスタム指標としてらくしふに連携されている:

```
GET /typed/api/admin/metrics/custom_fields
    ?store_id=945&genre_id=2&from=2026-07-19&to=2026-07-19&display_mode=day
```

- `results[]` に `{id, name, value_type, value_scope, term_mode, values}`
- `name:"修正客数"` (id17) / `name:"前年客数"` (id15)、`term_mode:"hour"`、`value_scope:"store"`
- `values[0].value` = 日合計（例 660）、`values[0].breakdowns[]` =
  `{start_minutes, end_minutes, value}` の60分刻み。360(=6:00)〜1800(=30:00)
- 2026-07-19 実測: 合計660、時間帯値が画面の修正客数行と完全一致
- **IDは会社依存のため名前でマッチする**（フィールド名は設定で変更可能に）
- store_id・genre_id はらくしふURLの `s=` / `g=` から取得（既存fetchActualと同じ）。
  store_scopeなのでgenre_idはどれでも同値（先頭の1つを使う）

見つからない場合（他社・未連携）: パネルにエラー表示＋利用可能なフィールド名一覧を
コンソールに出す（セットアップ支援）。

## 2. 計算モデル（負担率）

時間帯 h の LE を `LE[h]`（修正客数）とすると、F/K それぞれの負担込み客数:

```
load_G[h] = LE[h] + p1_G/100 × LE[h-1] + p2_G/100 × LE[h-2] + n1_G/100 × LE[h+1]
REQ_G[h]  = load_G[h] / y_G          （G ∈ {F, K}）
REQ計[h]  = REQ_F[h] + REQ_K[h]
```

- 係数は **F/K別々**（本人指定）。デフォルト: p1=50%（一時間前）、p2=0%（二時間前）、
  n1=0%（一時間後）、y=20名/人 — F/K同値スタート
- 範囲外の時間は breakdowns にある値を使う（h=23 の一時間後=24:00スロットは
  データにある）。無ければ0
- REQ表示は小数1桁（実計/不足行と同じ精度）。丸め（0.5単位切上げ等）は将来の
  設定候補として保留

**検算例（7/19実データ、デフォルト係数）:**
12時: LE=94, 11時LE=71 → load = 94 + 0.5×71 = 129.5 → REQ_F = REQ_K = 6.5人 → REQ計 13.0

## 3. 計算モード（設定でトグル）

| モード | LE | REQ | シート依存 |
|---|---|---|---|
| **らくしふ客数方式**（新規インストールのデフォルト） | 修正客数API | 負担率モデル計算 | 不要 |
| **シート方式**（現行動作） | シートLE行 | シートREQ行 | 必須 |

- らくしふ方式では設定なし（インストール直後）でもパネルが動く =
  配布版のゼロコンフィグ化。客数予測シートIDは「シート方式のときのみ必須」に変更
- らくしふ方式の表示:
  - 行: LE（修正客数）/ REQ F / REQ K / REQ計 ＋ 実F/実K/実FK/実計/不足（既存のまま）
  - REQ FK行は無し（モデルに概念が無い）。実FKは表示のみ（赤判定なし）
  - 上部チップ: LABOR%等（シート由来）の代わりに 修正客数計・前年客数計・前年比%
- タスクシートは両モードで独立（現状どおり任意）

## 4. 実装方針

`renderSheet()` のデータ取得部を分岐し、**らくしふ方式でも既存の
`hourly`（`extractSheetData`の出力形状: `{hours[], total}` per 行ラベル）に
詰め替える**。以降の表描画・不足計算・ヒートバー(v1.7)・実人数比較は無改修で動く。

```
fetchCustomerLe(date):
  GET metrics/custom_fields (URLのstore_id/先頭genre_id, from=to=date)
  → results.find(name === cfg.leFieldName)
  → breakdowns → le[h] (h=6..23) ＋ le24（23時の一時間後用）
  → キャッシュ（date単位、リロードボタンで破棄）

buildComputedHourly(le):
  hourly['LE']        = le
  hourly['REQ（F）']   = load_F/y_F
  hourly['REQ（K）']   = load_K/y_K
  hourly['REQ（SUM）'] = 和
  各 total も合計して同形状で返す
```

## 5. 設定画面の追加項目

「計算方法」セクション（新設）:

- 計算モード: ◉ らくしふ客数方式 / ○ スプレッドシート方式
- 客数フィールド名: `修正客数`（会社によって名前が違う場合に変更）
- 負担率（フロア）: 二時間前 `0`% ・一時間前 `50`% ・一時間後 `0`%
- 客数あたり人数（フロア）: `20` 名につき1人
- 負担率（キッチン）: 同上（別値）
- 客数あたり人数（キッチン）: `20` 名につき1人

storage キー追加（DEFAULTS拡張・すべて文字列）:
`calcMode('rakushifu'|'sheet', default 'rakushifu')`, `leFieldName('修正客数')`,
`fP2('0') fP1('50') fN1('0') fY('20')`, `kP2('0') kP1('50') kN1('0') kY('20')`

シート方式選択時のみ「客数予測シートID」を必須扱いにする（未設定パネルの文言も分岐）。
