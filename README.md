# らくしふ 客数予測パネル（配布版）

らくしふのシフト編集画面 (`/admin/v2/schedules`) に、時間帯別の客数(LE)・必要人数(REQ)・
実シフト人数・不足を重ねて表示するChrome拡張。

**v2.3から、らくしふの「修正客数」から必要人数を自動計算する「らくしふ客数方式」が
デフォルトになり、スプレッドシートなし・設定なしでそのまま動きます。**
店舗固有の値（負担率係数・genre分類・正社員名など）は設定画面から調整できます。

## 表示内容

- **上部チップ**: 修正客数計 / 前年客数計 / 前年比
  （シート方式では LABOR% / LABOR H / SALES / SBP）
- **時間帯テーブル** (6:00–23:00): LE(客数) / REQ F / REQ K / REQ計
  ＋ らくしふ実シフトから集計した実F / 実K / 実計（休憩控除済み）と不足行
  - 今日を表示中は現在時刻の列をハイライト、不足時間帯は赤表示
- **画面上の不足ヒートバー**: フロア/キッチン各セクションの時間軸直下に 実−REQ を表示
- **週間アサインバッジ**: スタッフ名の横に 週N日/Nh
- **タスク**: タスクシートから対象日に該当する月次/週次/要請タスクを表示
  （タスクシート未設定なら非表示）
- **シフト確定 未処理日**: 今日〜月末で「シフト確定」がまだの日を赤チップで列挙。
  📊ボタン（画面右上）にも件数バッジ表示。5分ごとに自動更新。

対象日はらくしふURLの `from=` に自動追従。パネルの ◀ ▶ で独立して前後の日も見られる。

## REQ（必要人数）の計算 — らくしふ客数方式

らくしふ日別ビュー上部の「修正客数」を LE とし、フロア(F)/キッチン(K)それぞれ:

```
負担込み客数[h] = LE[h] + 一時間前% × LE[h-1] + 二時間前% × LE[h-2] + 一時間後% × LE[h+1]
REQ[h] = 負担込み客数[h] ÷ 客数◯名につき1人
```

デフォルト: 一時間前50%・二時間前0%・一時間後0%・20名につき1人（F/K同値）。
すべて設定画面で店舗ごとに調整可能。
例: 12時LE=94、11時LE=71 → (94 + 71×0.5) ÷ 20 = **6.5人**。

## インストール（ストア不要）

1. このリポジトリを取得して展開
   - `git clone -b public https://github.com/AsukaIwanaga/RakushifuPanel.git`
   - または GitHubページで `public` ブランチを選び **Code → Download ZIP**
2. Chromeで `chrome://extensions` を開く
3. 右上「デベロッパーモード」をON
4. 「パッケージ化されていない拡張機能を読み込む」→ 展開したフォルダを選択
5. らくしふの編集画面を開く（またはリロード）→ 右上に📊ボタンが出る

**そのまま動きます**（らくしふ客数方式・デフォルト係数）。調整したい場合のみ次へ。

## 設定（任意・店舗ごと）

`chrome://extensions` → 本拡張の「詳細」→「拡張機能のオプション」:

1. **計算方法**: らくしふ客数方式（推奨）/ スプレッドシート方式
2. **負担率・必要人数**: F/K別に 二時間前%・一時間前%・一時間後%・◯名につき1人
3. **客数フィールド名**: 会社によって「修正客数」の行名が違う場合に変更
4. **genre_id**: フロア/キッチンのgenre_id。らくしふ編集画面でグループ絞り込みを
   フロアのみ/キッチンのみにしたときURLに付く `g=` の数字（例: `g=2` → 2）
5. **正社員の氏名**（任意）: MGT/TRer/TRee時間の計上先（MGT/cMGT）の振り分けに使用
6. **シート方式を使う場合**: 「時間帯別客数予測 v2」形式シートのURL（必須）と
   タスクシート（任意）

保存したら、らくしふのページを再読み込み。

## 仕組み

- 修正客数: らくしふ内部API `/typed/api/admin/metrics/custom_fields` から
  時間帯別breakdownsを取得（フィールドは名前でマッチ・店舗スコープ）
- シート取得（シート方式）: `docs.google.com` の gviz CSVエンドポイントを、ブラウザの
  GoogleログインCookieで取得（APIキー不要・シートの閲覧権限があれば動く）
- 実人数: らくしふ内部API `/ajax/admin/v2/schedules` から対象日のシフトを取得し、
  休憩控除・業務割振（F/K/FK振替、BU系→K、MGT/TRer/TRee→MGT系）込みで時間帯別に集計
- 未確定日: らくしふ内部API `shift_confirm_target_candidates` を利用。
  **レスポンス形式は防御的にパースしているため、初回は表示が実態と合っているか要確認。**

## 既知の注意点

- 「修正客数」フィールドが無い会社アカウントでは、パネルにエラーが出ます。
  DevToolsコンソールに利用可能なフィールド名一覧が出るので、設定の
  「客数フィールド名」を合わせてください
- らくしふ客数方式に REQ FK 行はありません（実FKは表示のみ・赤判定なし）
- シート方式: Googleからログアウトすると「シート取得失敗」→ Googleに再ログイン。
  シートの列配置（G列ラベル・6:00開始・J列ヘッダー値）はテンプレート準拠が前提
- 業務割振タスク名（F/K/FK/BU*/MGT/TRer/TRee）の解釈は共通ルールとしてコード内に固定。
  店舗で名前が違う場合は `content.js` の `moveGroup` を調整
- 設定を変更したら、らくしふページを再読み込み（自動リロードはしない）
- コードを更新したら `chrome://extensions` で ⟳（再読み込み）→ らくしふページもリロード
